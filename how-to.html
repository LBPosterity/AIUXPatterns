How to use the UI pattern references

These simplified UI examples are meant as a clarifying reference as you & your team evaluate common or new design patterns for AI interfaces.

Each pattern is not a complete workflow but is an abstracted example of a particular user experience design approach that could be adopted & combined with others to create an optimised AI experience for users.

Step 1

Have a clear goal for what success looks like for your design. Define the workflow & it's requirements.

Step 2

Review the patterns to see which one best fits your use case & user type.

Step 3

Use this reference, along with re world examples, to align on the approach for your bespoke design.

How LLM's are different

The majority of software utilise the common logic If > Then statements which are strung together to achive complex tasks. These allow us to code business logic into our products for reliable outcomes.

But...

LLM's (AI) are different, the natural language models aren't hard coded, they're more like a magical black box.

Although extremely powerful, the answers these AI models provide aren't always 100% accurate. The possibiliy of hallucinations (or lies) mean the AI output needs to be reviewed & interpreted by someone.

This will change as they become more reliable, but this is the key different when crafting AI user experiences.

Prompt abstraction types

The input for AI models are "simple" text prompt. While writing some text is easy enough, emedding a prompt with an effective amount of detail, content, specificity & direction is a skill. A skill the majority of users either are not capable of or do not have the time to author.

Like all user intefaces, we're abstracting a complex process or input to make it more intuitive & usable for people.

There 4 broad categories of rompt abstraction that potentially wrrant a specific UI. These types are:

  • Direct Prompt
    Users input text is directly fed to the AI model.
  • Supplemented Prompt
    The user prompt is embedded into existing prompt perameters.
  • Prompt Variables
    User selections influence variables in existing prompt peramters.
  • Variable Composition
    Larger prompt sections are conditionally included or omitted.

How do I use this guide?

Like all user interface patterns each one is optimised for a particular context or user type. It's important to have your individual requirements defined so you them to them to then assess what which pattern may work best for your product.
